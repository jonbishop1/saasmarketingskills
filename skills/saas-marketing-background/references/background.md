# Context: Intake Questions
 
Read before advising. Gather missing background rather than giving generic recommendations. Many questions have safe defaults for software — noted below so you don't ask for information the user doesn't have yet.
 
---
 
## Questions to Ask
 
**Goal**
- Primary objective? (trial signups, demo requests, API signups, paid conversions, lead gen, awareness)
- Target CPA or ROAS? *(Safe default: unknown until baseline established — channel-specific bidding strategy in each skill)*
- Monthly budget?
- Constraints? (geography, regulated industry, brand guidelines, compliance)
 
**Product**
- What's being promoted? (free trial, freemium, demo, paid plan, API access)
- Landing page URL?
- What makes the offer compelling to the target buyer?
- Regulated industry? (finance, healthcare, legal — affects ad copy and targeting restrictions)
 
**Audience**
- Ideal customer? (job title, company size, technical vs. non-technical buyer)
- Problem the product solves, in the customer's words?
- Target geography? *(Default: US only to start)*
 
**Current state**
- Other acquisition channels running? (important for attribution context)
- Existing landing page CVR data?
- CRM or sales pipeline?
 
---
 
## Safe Defaults (don't ask, just apply)
 
| Parameter | Default |
|---|---|
| Geography | United States to start |
| Primary conversion | Trial signup (PLG) or demo request (sales-led) |
 
Channel-specific defaults (bid strategy, budget sizing, device, placements, match types, attribution) live in each skill's own SKILL.md or guide.md.

## Routing After Intake
 
After gathering background, route to the appropriate skill:
 
| Situation | Where to go |
|---|---|
| Needs a marketing plan or strategy | `saas-marketing-plan` |
| Needs audience/market understanding first | `saas-audience-research` |
| Ready for Google Search Ads | `google-ads-for-saas` |
| Ready for Meta/Facebook/Instagram Ads | `meta-ads-for-saas` |
| Unsure which channel | Start with `saas-marketing-plan` for channel selection, then route to channel skills |
 
Each skill has its own channel-specific intake questions, safe defaults, and internal routing table. Check those after the shared background is gathered.
