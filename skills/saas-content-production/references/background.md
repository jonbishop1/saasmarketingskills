# Background: Content Production Intake Questions

Read this before advising. Gather missing background rather than giving generic recommendations. Many questions have safe defaults for software — noted below so you don't ask for information the user doesn't have yet.

---

## Questions to Ask

**What they need right now**
- Are they here to produce a specific piece of content, or to set up a production system?
- If a specific piece: what type? (blog post, guide, comparison, case study, docs, etc.)
- If a system: how many pieces per week/month are they targeting?
- Do they have a brief already, or do they need help creating one?

**Who writes**
- Who produces content currently? (Founder, co-founder, freelancer, agency, AI-assisted, nobody)
- If founder: how comfortable with long-form writing? *(Look for signals: "I write good docs" = strong foundation. "I hate writing" = coach the process, or help them hire.)*
- If freelancer/agency: what's the quality level? How's the current brief → review process?
- Budget for writers? *(Safe default: assume none unless stated — coach them to write themselves)*

**Content state**
- How many published pieces exist? *(Safe default: assume few or none)*
- Any content performing well? (Top pages by traffic or conversion)
- Any content they're unhappy with? Why?
- Do they have a style guide or brand voice documented? *(Safe default: no)*
- Do they have a keyword map or content calendar? *(If yes, use it. If no, reference `saas-content-strategy` for the strategic layer and `saas-seo-research` for keyword targeting.)*

**Production context (if producing a specific piece)**
- Target keyword? *(If none, reference `saas-seo-research`)*
- Target audience and their awareness level? (Problem-aware, solution-aware, product-aware)
- What should the reader do after reading? (CTA)
- Any existing content on the same topic they need to avoid cannibalizing?
- Deadline?
- Any specific examples, data, or product features to include?

**Repurposing**
- Do they repurpose content currently? *(Safe default: no)*
- What channels are they active on? (Twitter/X, LinkedIn, email newsletter, YouTube, podcast)
- How much time for repurposing per piece? *(Safe default: zero — only suggest repurposing if they have capacity)*

## Safe Defaults (don't ask, just apply)

| Parameter | Default |
|---|---|
| Writer | Founder writes, unless they say otherwise |
| Content type | Blog post, unless the situation suggests otherwise |
| Word count | Based on competitive SERP analysis, not a fixed number |
| Tone | Clear, direct, informed — like a senior engineer explaining to a peer |
| CTA | Soft product mention, not hard sell (unless it's a comparison/alternative page) |
| Publishing platform | Their existing blog/CMS |
| Repurposing | Skip unless they have time or ask about it |
| AI assistance | Coach them to use AI as draft-then-edit, never publish-as-is |
| Style guide | None — apply sensible defaults (active voice, short paragraphs, no jargon walls) |

| Needs short-form copy (headlines, CTAs, email subject lines) | Reference `saas-copywriting` skill |
